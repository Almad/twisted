Automate:

    ChangeLog

    QOTR

    SVN:
        New cycle: 
            cp svn://trunk svn://branches/wip-X.Y.Z
        New alpha:
            upver in wip-X.Y.Z
            cp svn://branches/wip-X.Y.Z svn://tags/release-X.Y.ZalphaN
            release()
        New rc1:
            upver in wip-X.Y.Z
            cp svn://branches/wip-X.Y.Z svn://tags/release-X.Y.Zrc1
            release()
        bugfixes:
            merge svn://branches/wip-X.Y.Z/FOO svn://trunk/FOO
            commit
        New rcN>1:
            upver in wip-X.Y.Z
            cp svn://branches/wip-X.Y.Z svn://tags/release-X.Y.ZrcN
            release()

    def release():
        export relevant tag
        generate documentation
        generate Plain, Docs, NoDocs tarballs
        generate Debian packages
        generate win32 packages
        distribute (tm.com and sourceforge)

        WebSite:
            update index
            products/download.html
            if final_release:
                products/get-current.epy
                documents/index.html
                documentation (/twisted/TwistedDocs/webdocs-gen.sh)
            cd /var/www/WebSite/twistedmatrix.com
            ./generate.sh

        if final_release:
            announce(fm, clpy, clpya, PyPI, LWN)

        notify packagers

So, the required input to './release-twisted' is:
    base-version
    type: (alphaN, rcN, final)
    optional: QOTR
    optional: files to merge for an rcN>1
